# 13 流式输出（Streaming）

本讲区分**非流式**与**流式**两种返回方式：非流式一次性返回完整回复，流式边生成边返回（如 SSE），可降低首字延迟、提升交互体验。

---

## 核心概念

### 1. 流式输出（Streaming）

**是什么**：服务端在生成回复时，不等到全部生成完毕再返回，而是每生成一小段（如一个 token 或一块）就立即推送给客户端。客户端可以边收边展示，形成「打字机」效果。

**为什么要有这个概念**：长回复时若等整段生成再返回，用户等待时间长、容易误以为卡死。流式能尽快展示第一个字，降低首字延迟，更适合对话、写作等场景。

### 2. 非流式（一次性返回）

**是什么**：客户端发请求后，服务端在内部完整生成回复，再一次性返回整段文本。客户端只能等请求结束后才能拿到完整内容。

**为什么要有这个概念**：非流式实现简单、便于做后处理（如一次性解析 JSON）；在对延迟不敏感或需要「完整结果再处理」的场景下仍常用。

### 3. SSE（Server-Sent Events）与 chunk

**是什么**：流式接口常采用 SSE 或类似「分块」协议：每块是一段文本或一个 token，按顺序推送。前端通过 EventSource 或 fetch + ReadableStream 按块读取并拼接展示。

**为什么要有这个概念**：理解「块」的边界与拼接方式，才能正确实现前端流式展示与后端流式 API 的对接。
