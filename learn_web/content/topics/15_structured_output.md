# 15 结构化输出（Structured Output）

本讲演示如何通过 **Prompt 约束**让模型输出**结构化内容**（如 JSON），再在代码中解析、校验，供下游程序使用。

---

## 核心概念

### 1. 结构化输出

**是什么**：不满足于「一段自由文本」，而是要求模型按约定格式输出，例如 JSON 对象、表格、列表。下游程序再解析该格式，做筛选、入库、展示或调用其他接口。

**为什么要有这个概念**：自由文本难以被程序稳定解析；结构化输出后可直接 `json.loads()`、按字段使用，减少正则或二次抽取的不可靠性。

### 2. Prompt 约束与格式约定

**是什么**：在 system 或 user 中明确写出「只输出一个 JSON」「字段名为 name、summary」「不要输出其他文字」等，引导模型按格式生成。也可配合 Few-shot 给一个示例。

**为什么要有这个概念**：模型默认不会主动输出严格格式；通过 prompt 约定是成本最低的「软约束」方式，无需微调或额外接口。

### 3. 解析与容错

**是什么**：模型可能多输出说明文字、或包在 markdown 代码块里。代码中需要从回复里**提取** JSON（如用正则取 `{...}` 或 ```json ... ```），再 `json.loads()`，并处理解析失败、缺字段等情况。

**为什么要有这个概念**：实际中模型未必 100% 遵守格式；解析与容错能提高鲁棒性，避免一次格式偏差就导致整段逻辑失败。

---

## 本 Demo 做了什么

### 基础用例

- 在 Prompt 中明确要求模型只输出符合约定格式的 JSON（如 `{"name": "...", "reason": "..."}` 或包含 `languages` 数组），调用 **core.llm** 后在代码中解析该 JSON（可从 markdown 代码块中提取），校验必要字段并打印。演示「格式约束 + 解析 + 校验」的下游可用输出。
- 运行：点击「运行基础 Demo」或执行 `python -m demos.15_structured_output.run`。

### 进阶用例

- 扩展为「多实体抽取」：要求模型从一段长文本中抽取多个实体，输出为 JSON 数组或嵌套结构，解析后遍历使用。
- 运行：点击「运行进阶 Demo」或执行 `python -m demos.15_structured_output.run_advanced`。
