# 项目重组与学习站规划

目标：将本项目明确为**大模型应用学习项目**，理顺目录、把 demo 挪出 apitest、将 skill 相关收拢为 demos/11_skill，并搭建**学习用 Web 站点**（每个 demo 对应知识点介绍 + 核心步骤演示）。

---

## 一、现状简要

| 位置 | 内容 |
|------|------|
| 根目录 | OpenClaw 介绍、package.json、demo_agent.py、requirements.txt、INSTALL_OPENCLAW.md |
| `apitest/` | 统一 LLM（llm/）、chat_app、rag_doc、mcp_demo、以及 8 个主题 demo（function_calling、prompt、memory、agent、embedding_rag、eval、multimodal） |
| `skills/` | demo-skill（OpenClaw SKILL.md 示例） |
| `docs/` | 大模型应用学习路径.md |

问题：demo 与「API 测试」混在 apitest 下；skill 与「大模型学习」关系不直观；缺少统一的学习入口（Web）。

- **learn_web**：单一大模型学习站点，首页=学习路径，子页=各 demo 的「知识点 + 演示」。

## 五、学习站 Web（learn_web）功能规划

### 5.1 页面结构

1. **首页（/）**
   - 标题：大模型应用学习
   - 按学习路径列出 10 个主题（0. LLM 调用 … 10. 多模态），每项链接到对应 demo 页。

2. **各 Demo 页（/demo/01_api_chat … /demo/10_multimodal）**
   - **知识点介绍**：左侧或上方固定区域，展示该主题的说明（来源：`docs/大模型应用学习路径.md` 或 `learn_web/content/topics.json`）。
   - **核心执行步骤演示**：右侧或下方，可选两种方式：
     - **方式 A**：页内「运行 Demo」按钮，调用后端执行对应 demo 的入口（如 `demos/04_function_calling/run.py` 的 main），将输出以日志/步骤形式展示在页面上。
     - **方式 B**：分步展示「步骤 1 / 2 / 3…」的说明 + 每步的示例输入/输出（静态或从后端拉取预跑结果），不真实调 LLM，仅作教学。
   - 建议先实现**方式 A**（真实执行），便于学习者直接看到效果；方式 B 可作为「步骤说明」与方式 A 并存。

### 5.2 后端接口（示例）

- `GET /api/topics`：返回各主题 id、标题、简介、顺序（供首页与导航用）。
- `GET /api/demo/<topic_id>/intro`：返回该 demo 的完整知识点文案（Markdown 或 HTML）。
- `POST /api/demo/<topic_id>/run`：执行该 demo 的入口函数，返回步骤化输出（如 `[{step, message, result}]`）；可选带参数（如 prompt_demo 的「问题」、memory_demo 的「是否加载历史」）。
- 静态资源：CSS/JS 放在 `learn_web/static/`，模板用 `learn_web/templates/`。

### 5.3 技术选型

- **后端**：Flask（与现有 chat_app 一致，易复用 core/llm）。
- **前端**：单页或简单多页均可；知识点区域可用 Markdown 渲染（如 marked 或后端转 HTML）。
- **执行 demo**：通过 `importlib` 或 `subprocess` 调用 `demos/xx/run.py`，捕获 stdout/stderr 或封装为「步骤」列表返回；注意超时与进程隔离。

## 六、实施顺序建议

| 阶段 | 内容 | 说明 |
|------|------|------|
| 1 | 建 `core/`，迁入 llm、rag_doc | 保持原 apitest 的 llm/rag_doc 行为，仅改路径与 import |
| 2 | 建 `demos/`，迁入 10 个主题 demo | 每个目录下保留 run.py、README；批量改 import 为 core |
| 3 | 建 `demos/11_skill/`，迁入 skills/demo-skill，补 README | 链接与文档指向 demos/11_skill |
| 4 | 收尾 apitest | 删除已迁移内容；chat_app 迁入 learn_web 或保留 core 一处入口 |
| 5 | 建 `learn_web/` 骨架 | app.py、首页、10 个 demo 子页占位、topics 数据来源 |
| 6 | 实现知识点展示 | 从 docs 或 content 拉取各主题介绍并渲染到对应页 |
| 7 | 实现「运行 Demo」与步骤展示 | 后端调用 demos/xx 执行并返回步骤/日志，前端展示 |
| 8 | 文档与 README 更新 | 根 README、docs/大模型应用学习路径.md 中的路径与链接全部指向新结构 |

## 七、可选简化

- 若希望**改动最小**：可暂不建 `core/`，保留 `apitest/` 仅放 llm 与 rag_doc，只把「demo 目录」挪到根目录 `demos/`，learn_web 和 demos 仍从 `apitest` 引用 llm/rag_doc。
- 若希望**一步到位**：按第二节完整迁移到 core + demos（含 11_skill）+ learn_web，并统一从项目根运行学习站与各 demo。
