# 大模型应用学习路径

本仓库已包含：**API 调用**、**联网检索（类 RAG）**、**MCP 工具调用**，以及按主题拆分的独立 demo 目录。所有 LLM 调用统一走 **`core/llm`** 模块（火山方舟 Ark API）。Demo 位于 **`demos/`**，学习站 Web 为 **`learn_web/`**（<http://127.0.0.1:5001/>）。✅ 表示本仓库已有对应 demo，➡️ 表示建议下一步。

---

## 0. 统一 LLM 调用模块 ✅

- **内容**：封装火山方舟 API，供各主题 demo 与对话应用复用；配置优先 .env / 环境变量，见 [core/llm/README.md](../core/llm/README.md)。
- **本仓库**：**`core/llm/`**（`config`、`client`、`chat`/`chat_stream`）。参见 [core/llm/README.md](../core/llm/README.md)。

---

## 1. 基础：API 与对话 ✅

- **内容**：调用大模型 API、发消息、收回复；流式 / 非流式；多轮对话（messages 历史）。
- **本仓库**：`demos/01_api_chat`、`learn_web`；综合对话应用见 `demos/12_chat_app`。
- **延伸**：换其他 API（OpenAI、Claude、国产大模型），在 `llm` 内增加适配或新 client。

---

## 2. 检索增强（RAG）✅

- **内容**：用户问题 → 先检索相关文档/网页 → 把检索结果和问题一起交给大模型 → 生成有依据的答案。
- **本仓库**：**`demos/02_rag`**、**`core/rag_doc`**（文档 RAG）；原 chat_app 仍支持联网搜索与文档 RAG。
- **延伸**：用 Embedding + 向量库替代 TF-IDF；可选框架 LangChain / LlamaIndex。

---

## 3. 工具调用（MCP）✅

- **内容**：大模型需要「查天气、查库、发邮件」等能力时，通过标准协议调用外部工具；服务端暴露工具列表（list_tools）与执行（call_tool），客户端发起 list 再按需 call。
- **本仓库**：**`demos/03_mcp/`**（天气 get_weather、server、list_tools_client）；Cursor 见 `.cursor/mcp.json`（args: `demos.03_mcp.server`）。
- **延伸**：多工具、多 MCP Server；让 LLM 根据用户意图选工具、填参数（可配合 Function Calling 或 Agent）。

---

## 4. Function Calling（模型原生工具调用）✅

- **内容**：模型在回复里直接返回「要调用的函数名 + 参数」；应用解析后执行函数，把结果再塞回对话，让模型继续生成。
- **本仓库**：**`demos/04_function_calling/`**。用 Prompt 约定让模型输出 `TOOL: 函数名 参数`，解析后执行再交给 LLM。运行：`python -m demos.04_function_calling.run`。
- **建议**：在 API 支持时改为传 `tools`、解析响应里的 `tool_calls`。

---

## 5. Prompt 工程 ✅

- **内容**：System prompt 设计、角色设定、Few-shot 示例、Chain-of-Thought、输出格式约束（JSON/表格等）。
- **本仓库**：**`demos/05_prompt/`**。同一问题用不同 system prompt，对比输出。运行：`python -m demos.05_prompt.run`。
- **延伸**：Few-shot 示例、CoT、结构化输出（JSON schema）。

---

## 6. 多轮对话与记忆 ✅

- **内容**：维护对话历史（窗口长度、截断策略）、可选摘要压缩、会话隔离（按 user/session 存）。
- **本仓库**：**`demos/06_memory/`**。已实现：会话持久化 memory.md、超长摘要、长期记忆 RAG。运行：`python -m demos.06_memory.run`。
- **延伸**：多 Session 管理、Redis/DB 持久化、记忆去重与过期。

---

## 7. Agent / 智能体（多步推理与规划）✅

- **内容**：模型自主决定「先查什么、再查什么、最后回答」；ReAct（Thought–Action–Observation）或 Plan-and-Execute；可结合 MCP/Function Calling。
- **本仓库**：**`demos/07_agent/`**。已实现：基础 `run.py`、ReAct `run_react.py`、多 MCP `run_multi_mcp.py`、规划拆解 `run_plan.py`。运行见 [demos/07_agent/README.md](../demos/07_agent/README.md)。
- **延伸**：接入真实 MCP 进程、多轮规划与回溯。

---

## 8. Embedding 与向量检索（RAG 深化）✅

- **内容**：用 Embedding API 把文本变成向量；存入向量库（Milvus、Chroma、FAISS、pgvector 等）；问句向量化后检索相似片段，再交给 LLM。
- **本仓库**：**`demos/08_embedding_rag/`**。用 TF-IDF 模拟向量检索，检索后交给 core.llm；生产可换 Embedding API + 向量库。运行：`python -m demos.08_embedding_rag.run`。
- **建议**：接入真实 Embedding 与向量库，与文档 RAG 统一为「向量检索 RAG」。

---

## 9. 评估与评测（Eval）✅

- **内容**：对回答做自动评测（相关性、事实性、格式）；构造测试集、跑批量请求、记分与对比。
- **本仓库**：**`demos/09_eval/`**。若干（问题, 期望关键词），LLM 作答后用关键词判分。运行：`python -m demos.09_eval.run`。
- **延伸**：LLM-as-judge、与标准答案相似度、多维度打分。

---

## 10. 多模态（图/音）✅

- **内容**：支持图像/语音输入或输出（Vision API、TTS/STT）；多模态 RAG（图+文检索与生成）。
- **本仓库**：**`demos/10_multimodal/`**。演示多模态消息结构与文本调用。运行：`python -m demos.10_multimodal.run`。
- **建议**：以火山方舟多模态文档为准，接入 Vision/语音 API。

---

## 11. OpenClaw Skill 示例 ✅

- **内容**：OpenClaw Skill 规范与 SKILL.md 写法；将自定义能力注入 Gateway/Agent。
- **本仓库**：**`demos/11_skill/`**。示例 demo-skill；若需被 OpenClaw 加载，复制到项目根 `skills/`。

---

## 12. 综合对话应用 ✅

- **内容**：将 RAG、MCP、联网搜索整合在一个 Web 对话应用中，可勾选启用各工具；流式回复。
- **本仓库**：**`demos/12_chat_app/`**。运行：`python -m demos.12_chat_app.app`，访问 http://127.0.0.1:5000。

---

## 13. 流式输出（Streaming）✅

- **内容**：流式与非流式对比；边生成边返回（逐 token/chunk），降低首字延迟、提升对话体验；SSE 与前端消费方式。
- **本仓库**：**`demos/13_streaming/`**。同一问题先非流式再流式调用，演示逐 token 打印。运行：`python -m demos.13_streaming.run`。
- **延伸**：与 01 基础 API 配合理解；前端 SSE/ReadableStream 对接。

---

## 14. 结构化输出（Structured Output）✅

- **内容**：通过 prompt 约束让模型输出 JSON 等固定格式；在代码中解析、校验，供下游程序使用；解析容错（从 markdown/说明文字中提取 JSON）。
- **本仓库**：**`demos/14_structured_output/`**。要求模型只输出指定 JSON 结构并解析。运行：`python -m demos.14_structured_output.run`。
- **延伸**：与 05 Prompt 工程中的「输出格式约束」结合；API 若支持 JSON schema 可进一步约束。

---

## 15. Temperature 与采样参数 ✅

- **内容**：Temperature 控制采样随机性：低 temperature 输出更稳定、重复性高，高 temperature 更多样、更有创造性；同一问题不同参数对比。
- **本仓库**：**`demos/15_temperature/`**。同一问题分别用低/高 temperature 调用并对比输出。运行：`python -m demos.15_temperature.run`。
- **说明**：若当前 API 不支持 temperature 可能报错，需查阅接口文档。

---

## 16. 错误处理与重试 ✅

- **内容**：对 LLM 请求做重试与指数退避，应对瞬时故障（超时、限流、5xx）；可重试与不可重试错误的区分。
- **本仓库**：**`demos/16_error_retry/`**。封装带重试的 chat，演示失败后退避再试。运行：`python -m demos.16_error_retry.run`。
- **延伸**：与限流、熔断、降级策略结合。

---

## 17. Token 与成本意识 ✅

- **内容**：上下文以 token 计费；用简单启发式估算「消息字符数 → 约 token 数」，建立长度与成本概念；生产宜用 tokenizer 或 API 返回的 usage。
- **本仓库**：**`demos/17_tokens/`**。对示例消息做字符数与约 token 估算（不调用 LLM）。运行：`python -m demos.17_tokens.run`。
- **延伸**：按 token 限流、预算控制、长上下文成本优化。

---

## 18. 上下文截断（Context Truncation）✅

- **内容**：对话历史超长时只保留「最近 N 条」再发给模型，避免超上下文窗口报错或成本过高；截断策略与摘要压缩的配合（见 06 记忆）。
- **本仓库**：**`demos/18_context_truncate/`**。构造模拟长历史，按「保留最近 K 条」截断后请求。运行：`python -m demos.18_context_truncate.run`。
- **延伸**：按 token 数截断、保留首条 system + 最近 N 轮、结合摘要。
